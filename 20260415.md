## 1. 왜 이 회의를 여는가

4/13 회의에서 결정된 큰 프레임(대흥 10곳 직접 미팅 + 인스타 광고 2안)은 유효하다. 그러나 48시간 동안 다음 3가지가 선명해졌다:

1. **데이터 집약 지점이 없다** — 영업 미팅·인스타 광고·무료체험 신청이 각각 따로 움직이면 4/30에 의사결정용 데이터가 파편화된다. **통합 랜딩페이지 = 모든 채널이 꽂히는 단일 응답 수집기**가 필요하다.
2. **4월 전체를 관통하는 최우선 산출물이 모호하다** — "시장 검증"이라는 말은 넓다. 4월 말까지 **고객 니즈 / 요금제 / 서비스 이름 / 핵심 기능 Top 3 / 핵심 유저 플로우** 5가지를 확정하는 것이 진짜 목표다. 이것이 5월 MVP 배포의 기반이다.
3. **QA가 이벤트성이면 MVP가 무너진다** — 동주의 QA는 1·2차 리포트가 아니라 **엔드리스 QA(매일 반복 + 실시간 피드백)** 로 전환해야 한다. 이탈리아 원격 개발은 시차가 있으므로, 동주의 피드백이 주연의 밤 시간 푸시에 즉시 반영되는 루프가 필요하다.

## 2. 4월 목표 재정의 (반드시 확정해야 할 5가지)

4월은 MVP를 "철저하게" 완성하는 달이다. 뇌피셜 금지. 마케팅(선화) + 영업(재영) 데이터로 아래 5가지를 확정한다.

| # | 산출물 | 확정 기준 | 데이터 소스 |
|---|--------|-----------|-------------|
| 1 | **고객 니즈** | 5개 기능 × 10곳 반응 점수 평균 + 광고 반응률 | 재영 미팅 + 선화 광고 |
| 2 | **서비스 이름** | 후보 3~5개 → 인스타 광고 A/B 노출로 CTR 검증 | 선화 광고 CTR |
| 3 | **요금제** | 독점/크레딧/하이브리드 중 1개 확정 | 재영 구매의향 + 사전예약 전환율 |
| 4 | **핵심 기능 Top 3** | 현재 40+ 기능 목록 중 반응 1~5점 평균 4점 이상 3개 | 재영 기능별 점수 기록 |
| 5 | **핵심 유저 플로우** | 원장 하루 시나리오 → 앱 진입부터 시험지 출력까지 ≤ 7클릭 | 동주 QA + 재영 현장 관찰 |

**5월 목표:** 이 5가지를 근거로 실사용자(사전예약자 + 무료체험 신청자)에게 MVP를 배포. 스케일업 시작.

## 3. 김주연 (CTO) 업데이트 — 통합 랜딩페이지 제작 추가

### 왜 랜딩페이지가 필요한가

- 인스타 광고의 CTA(무료 체험 / 사전 예약)가 꽂힐 단일 URL이 없다면, 광고 예산이 전환 없이 소진된다.
- 영업 미팅 후 "생각해볼게요" 고객을 데려갈 수 있는 2차 공간이 없다.
- **랜딩페이지 = 4월 내 수집되는 모든 정성/정량 데이터의 단일 저장소.**

### 랜딩페이지 섹션 구조 (4/15 확정, 4/16~19 구축)

1. **히어로** — 서비스 이름 A/B (예: "에듀플로우" vs "틴북" vs "원장AI") × "우리 구에 딱 3자리" 프레이밍
2. **핵심 기능 데모 영상** — 지문 업로드 → AI 시험지 생성 20초 영상
3. **기능 5종 상세** — ERP / AI 지문분석 / AI 문제생성 / 자동조판 / 학습리포트
4. **가격 3안 비교 테이블** — 독점 월 50만 / 크레딧 / 하이브리드 (어느 안에 클릭 가장 많이 오는지 측정)
5. **CTA 분기** — [무료 체험 신청] | [사전 예약(50만원)] — 두 버튼 모두 노출, 클릭 비율로 의향 측정
6. **응답 폼(필수 필드 최소화)** — 학원명 / 지역(구/동) / 규모 / 주요과목 / 연락처 / 희망 미팅일 / 현재 시험지 제작 방식 / 관심 기능(체크박스)

### 응답 필드 = 측정하고 싶은 가설 리스트

| 필드 | 측정하고 싶은 것 |
|------|------------------|
| 학원 위치(구/동) | 어느 지역 수요가 집중되는가 (대흥 vs 타지역) |
| 규모(학생 수) | 어느 규모 학원이 가장 민감한가 |
| 주요 과목 | 영어 외 과목 확장 우선순위 |
| 현재 시험지 제작 방식 | 경쟁 제품/수기 비중 |
| 관심 기능(5개 체크박스) | 기능별 선호도 정량화 |
| CTA 클릭(무료체험 vs 사전예약) | 가격 저항 지점 |
| 이름 A/B 노출 분기 | 서비스 이름 확정 근거 |

### 기술 스택
- Next.js + Tailwind + Vercel 호스팅
- 응답: Supabase(DB) + 노션 임베드 미러링 (임원진 실시간 확인용)
- 광고 UTM 태그별 전환 추적

- 컨셉: "AI가 시험지를 만들어준다? 선생님, 직접 체험해보세요"

- 컨셉: "우리 구에 딱 3자리. 독점 이용권 사전 예약"
- CTA 링크: 랜딩페이지 `/reserve` → 예약 폼
- 일 예산 2~3만원 × 8일

```
인스타 광고(1안+2안)
```

## 6. 이동주 (재무/운영) — 엔드리스 QA로 전환

### "엔드리스 QA"의 의미
- 기존: 1차 리포트(4/18) + 중간(4/25) + 최종(4/30) — 3이벤트.
- 변경: **매일 QA 사이클 실행**. 주연의 푸시가 올 때마다 동일 날짜 내 검증.

### 일일 QA 사이클

```
09:00 주연의 이탈리아 밤 푸시 확인
10:00~12:00 신규 기능 QA
13:00 재영 미팅 인사이트 수신 → QA 항목 변환
14:00~16:00 문제 퀄리티 검수
16:00 선화 광고 댓글·문의 모니터링 → 개선안
17:00 노션 일일 리포트 작성 → 주연에게 푸시
```

### 실시간 피드백 루프
- 노션 "QA 이슈 보드": Critical / Major / Minor 태그
- Critical 이슈 발견 시 즉시 슬랙 주연 멘션
- 주연 이탈리아 낮(한국 밤) 시간에 수정 푸시
- 다음날 아침 동주 재검증

### QA 대시보드 (주연·재영이 이탈리아에서 확인)
- 전체 이슈 수 / 해결률 / 평균 해결 시간
- 기능별 이슈 분포
- 재영 인사이트 → QA 전환율
- 선화 광고 피드백 → 개선안 전환율

## 7. 5월 스케일업 플랜

### 5월 = 실사용자 배포
4월 MVP 완성 → 5월 1주차 사전예약자·체험신청자에게 실제 배포.

### 5월 주차별 계획

| 주차 | 핵심 |
|------|------|
| 5/1주 | 사전예약자 10~20명 우선 온보딩. 매일 사용 데이터 수집. |

| 5/4주 | 대흥 외 지역 확장. 1개월 MRR 측정. |

### 5월 리스크
- 온보딩 이탈률 높을 시: UI/UX 긴급 리웍 (동주 QA 데이터 근거)
- 유료 전환율 < 10%: 가격 재설정 or 크레딧 모델 전환
- 기능 요청 폭주: 핵심 기능 Top 3 외 전부 5월 후순위
